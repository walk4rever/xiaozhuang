# 小庄 — 帮你找到最贴切的那句话

小庄是一个面向现代情绪与表达场景的中文灵感产品。

当你**说不清、想不明、写不出**的时候，小庄借助中国古典语言与传统智慧，帮你找到一句更能安放此刻心境的话。

> 借古人的话，说今天的心。  
> 天地有大美而不言 —— 庄子

---

## 产品定位

小庄不是泛 AI 助手，也不是古文工具箱。

它更准确的定位是：

> **一个帮助现代人借古人的语言，说出今天的心的产品。**

它解决的不是“查一句古诗”这么简单的问题，而是：
- 我有一种感觉，但表达不出来
- 我想发点什么，但没有合适的话
- 我想安放一个情绪，却找不到语言
- 我有一点困惑，想得到一个更有意味的回应

当前阶段，小庄的主线是：

- **表达**：帮用户说出
- **照见**：帮用户想明白

## 当前功能与产品结构

### 🌅 寻句（主产品）
描述一个场景、一种情绪，或者直接上传一张照片，小庄会从中国古典诗词、古文中找到最贴切的一句，并给出：
- 原句
- 出处
- 白话解读
- 为什么它适合你此刻的场景

这是小庄当前最核心、最日常、最适合反复使用的能力。

### 🔮 问卦（特色功能）
当你心里有疑惑、有犹豫、有想不明白的事时，可以起一卦。小庄结合周易卦象与 AI 解读，给你一个更有意味的回应。

问卦是小庄的重要品牌特色，但不是当前唯一主线。

### ✍️ 仿写（表达延展）
仿写不是单独的大产品，而是承接寻句之后的表达动作：

> 找到一句之后，进一步把它写成适合“我此刻”的一句。

### 📮 慢读（长期陪伴）
慢读不是古文工具页，而是：
- 每日一封
- 统一大课
- 像来信，不像讲义
- 用古文照见现代人的情绪与处境

一句话：

> 寻句负责即时使用，慢读负责长期陪伴。

## 近期优先功能

### 1. 微信小程序主入口
这是接下来最高优先级的产品动作。

原因很直接：
- 更符合真实中文用户的使用环境
- 更适合承接拍照、分享与回流
- 更适合情绪触发型、轻量高频型使用场景

目标是让小庄从“可访问的网站”变成“更适合日常打开的产品”。

### 2. 慢读大课版
慢读第一版当前已经明确：
- 只做统一大课
- 不做小课订阅
- 不做用户指定作者 / 书名 / 人物名
- 内容按现代情绪 / 处境组织
- 写法强调“来信感”而不是“课程感”

要验证的核心不是个性化，而是：

> 用户会不会因为“这是小庄今天发来的一封信”而持续打开。

## 本地开发

```bash
npm install
npm run dev
```

环境变量（`.env.local`）：

```bash
AI_API_KEY=your-api-key
AI_API_BASE_URL=https://your-provider.example.com/v1
AI_PRIMARY_MODEL=your-text-model
AI_VISION_MODEL=your-vision-model   # 推荐单独配置，供 image / video 请求使用

SUPABASE_URL=https://your-project.supabase.co
SUPABASE_SERVICE_ROLE_KEY=your-supabase-service-role-key
RESEND_API_KEY=your-resend-api-key
DU_FROM_EMAIL=du@example.com
CRON_SECRET=your-cron-secret
```

说明：
- `AI_PRIMARY_MODEL`：默认/主文本模型，用于寻句、问卦等纯文本能力
- `AI_VISION_MODEL`：单独的视觉模型；当请求里包含 image / video 输入时优先使用
- 若未配置 `AI_VISION_MODEL`，多模态请求会回退到 `AI_PRIMARY_MODEL`
- `SUPABASE_URL` / `SUPABASE_SERVICE_ROLE_KEY`：慢读订阅、退订、每日发送记录所需
- `RESEND_API_KEY`：用于发送每日慢读邮件
- `DU_FROM_EMAIL`：慢读邮件发件人地址
- `CRON_SECRET`：保护 `/api/du/cron/send` 定时任务入口
