## Part B：慢读（替代“秒读”的新的 P1）

## 新定义

“秒读”改名为：

> **慢读**

产品核心不是“30 秒读懂”，而是：
- 每日一封
- 慢慢理解
- 长期积累
- 持续建立与用户的关系

更具体地说，慢读第一版的目标不是个性化，而是：

> **验证用户会不会稳定打开由小庄统一发送的每日来信。**

因此当前阶段明确：
- **只做统一大课**
- **不做小课订阅**
- **不做用户指定作者 / 书名 / 人物名的不同课程流**

先把“大课版慢读”跑通，再考虑未来是否需要更细的主题偏好。

## 慢读大课 v1：产品原则

### 核心产品形态
- [ ] 所有订阅用户每天收到同一封邮件
- [ ] 邮件内容围绕同一个当日主题展开
- [ ] 主题优先来自现代人的情绪与处境，而不是古籍分类
- [ ] 让用户形成“今天小庄又来了一封信”的期待感

### 当前明确不做
- [x] 不开放用户自由指定作者 / 书名 / 人物名
- [x] 不做每人不同内容流
- [x] 不做复杂的小课订阅系统
- [x] 不把慢读做成内容平台或知识工具箱

### 大课选题方向
- [ ] 焦虑时读什么
- [ ] 想把事情做长一点时读什么
- [ ] 面对关系时读什么
- [ ] 想念某个人时读什么
- [ ] 面对得失时读什么
- [ ] 春天、雨天、深夜、离别、重逢时读什么

原则：

> 古文是材料，古人是引路人，现代人的处境才是目录。
